## 1. プロジェクト概要

### 1.1 基本情報

| 項目 | 内容 |
|------|------|
| プロジェクト名 | FieldBase-K ウェブサイト構築 |
| 事業内容 | キャンピングカーレンタル事業（1台運営） |
| ドメイン | fieldbase-k.jp（取得済み） |
| 想定公開時期 | 要調整 |

### 1.2 プロジェクトの背景

本プロジェクトは、FieldBase-Kが運営するキャンピングカーレンタル事業の認知拡大と集客を目的としたウェブサイトの新規構築である。

キャンピングカー1台という小規模運営の特性を活かし、きめ細やかなサービス訴求と運用効率の両立を実現する。

### 1.3 設計原則

本プロジェクトは以下の原則に基づいて設計・実装を行う。

1. **シンプルさの原則**: 1台運営という規模に最適化し、過剰な機能を排除する
2. **運用負荷最小化の原則**: オーナー自身で更新・管理可能な構成とする
3. **SEOファーストの原則**: 検索エンジンからの自然流入を最大化する
4. **パフォーマンス優先の原則**: Core Web Vitals を満たす高速なサイトを構築する
5. **漸進的拡張の原則**: 将来的な機能追加に対応可能なアーキテクチャとする

## 2. 目的・目標

### 2.1 事業目標

1. キャンピングカーレンタルの認知度向上
2. ウェブサイト経由での問い合わせ獲得
3. 予約率の向上（空き状況の可視化による）
4. ブランドイメージの確立

### 2.2 ウェブサイト目標

| 目標カテゴリ | 指標 | 目標値 |
|-------------|------|--------|
| 集客 | 月間オーガニック訪問数 | 公開後6ヶ月で500PV/月 |
| コンバージョン | 問い合わせ数 | 月間5件以上 |
| パフォーマンス | Core Web Vitals | 全指標「良好」 |
| SEO | 地域キーワード検索順位 | 10位以内 |

### 2.3 KPI定義

1. 問い合わせ完了率 = 問い合わせ完了数 / 問い合わせページ訪問数
2. 直帰率 < 50%
3. 平均セッション時間 > 2分
4. カレンダーページ閲覧率 > 30%（サイト訪問者の30%以上がカレンダーを確認）

## 4. 機能要件

### 4.1 サイト構成

| ページ | URL | 概要 |
|--------|-----|------|
| ホーム | / | ファーストビュー、魅力訴求、各ページへの導線 |
| 車両詳細 | /vehicle | キャンピングカーのスペック、設備、写真ギャラリー |
| 料金 | /pricing | 料金表、シーズン別料金、オプション |
| ご利用の流れ | /flow | 予約から返却までのステップ解説 |
| 空き状況 | /calendar | 予約可能日をカレンダー形式で表示 |
| お問い合わせ | /contact | 問い合わせフォーム |
| FAQ | /faq | よくある質問と回答 |

### 4.2 各ページ詳細仕様

#### 4.2.1 ホームページ（/）

**【目的】** サイトの顔として、事業の魅力を伝え、各ページへ誘導する

**【必須要素】**

1. **ヒーローセクション**: キャンピングカーの魅力的な写真、キャッチコピー、CTAボタン
2. **サービス概要**: 3〜4つの特徴をアイコン付きで簡潔に紹介
3. **車両プレビュー**: 車両写真と「詳しく見る」ボタン
4. **料金プレビュー**: 基本料金の表示と「料金詳細」ボタン
5. **お客様の声（任意）**: レビューや体験談
6. **CTAセクション**: 問い合わせ・空き状況確認への誘導

#### 4.2.2 車両詳細ページ（/vehicle）

**【目的】** キャンピングカーの詳細情報を提供し、利用イメージを具体化する

**【必須要素】**

1. **車両基本情報**: 車種名、乗車定員、就寝定員、サイズ
2. **写真ギャラリー**: 外観、内装、設備の写真（10枚以上推奨）
3. **設備一覧**: キッチン、冷蔵庫、エアコン、寝具等の装備リスト
4. **スペック表**: 全長/全幅/全高、排気量、燃費等
5. **注意事項**: 運転に必要な免許、制限事項

#### 4.2.3 料金ページ（/pricing）

**【目的】** 明確な料金体系を提示し、予約検討を促進する

**【必須要素】**

1. **基本料金表**: 1泊2日、2泊3日等の料金
2. **シーズン料金**: 通常期、繁忙期、オフシーズンの区分と料金差
3. **オプション料金**: 追加備品、保険等
4. **料金に含まれるもの/含まれないもの**
5. **キャンセルポリシー**

#### 4.2.4 空き状況カレンダー（/calendar）

**【目的】** リアルタイムの空き状況を可視化し、予約検討を促進する

**【実装方針】**

1. Google Calendar の iframe 埋め込みを推奨（運用負荷最小化の原則）
2. 代替案: カスタム実装（Notion API / Google Sheets API連携）

**【表示要件】**

- 予約済み/空きの状態が一目で分かる配色
- スマートフォンでも操作しやすいUI
- 月表示を基本とし、前後月への移動が可能

#### 4.2.5 お問い合わせページ（/contact）

**【目的】** 見込み顧客からの問い合わせを獲得する

**【フォーム項目】**

1. 氏名（必須）
2. メールアドレス（必須）
3. 電話番号（任意）
4. 利用希望日（任意・日付選択）
5. 利用日数（任意・プルダウン）
6. お問い合わせ内容（必須・テキストエリア）

**【実装方針】** Formspree または Netlify Forms を推奨

**【要件】**

- 送信完了後のサンクスページ表示
- 管理者へのメール通知
- スパム対策（reCAPTCHA等）
- 入力バリデーション

#### 4.2.6 ご利用の流れページ（/flow）

**【目的】** 予約から返却までのプロセスを明確にし、不安を解消する

**【必須要素】**

1. **ステップ形式の説明**: お問い合わせ → 予約確定 → 受取 → 利用 → 返却
2. **各ステップの詳細**: 所要時間、必要なもの、注意点
3. **よくある質問へのリンク**

#### 4.2.7 FAQページ（/faq）

**【目的】** ユーザーの疑問を事前に解消し、問い合わせの質を向上させる

**【必須カテゴリ】**

- 予約について
- 料金について
- 車両について
- 利用当日について
- キャンセルについて

**【SEO観点】** FAQ構造化データ（FAQPage schema）を実装

## 5. 非機能要件

### 5.1 パフォーマンス要件

| 指標 | 目標値 | 測定方法 |
|------|--------|----------|
| LCP (Largest Contentful Paint) | 2.5秒以下 | PageSpeed Insights |
| FID (First Input Delay) | 100ms以下 | PageSpeed Insights |
| CLS (Cumulative Layout Shift) | 0.1以下 | PageSpeed Insights |
| TTFB (Time to First Byte) | 600ms以下 | WebPageTest |
| PageSpeed スコア（モバイル） | 90以上 | PageSpeed Insights |

### 5.2 セキュリティ要件

- HTTPS必須（SSL/TLS証明書の適用）
- フォーム送信時のCSRF対策
- XSS（クロスサイトスクリプティング）対策
- 依存パッケージの脆弱性スキャン

### 5.3 可用性要件

- 稼働率: 99.9%以上（ホスティングサービスのSLAに準拠）
- CDN配信による高可用性確保

### 5.4 互換性要件

**【対応ブラウザ】**

- Chrome（最新版）
- Safari（最新版）
- Firefox（最新版）
- Edge（最新版）

**【対応デバイス】**

- デスクトップ: 1280px以上
- タブレット: 768px〜1279px
- スマートフォン: 320px〜767px

**【レスポンシブ対応】** モバイルファーストで設計

## 7. SEO要件

### 7.1 技術的SEO

1. **メタタグ最適化**: 各ページ固有のtitle、description設定
2. **OGP設定**: Facebook、Twitter用のOGタグ設定
3. **構造化データ**: LocalBusiness、Product、FAQPageスキーマの実装
4. **サイトマップ**: XML形式、自動生成（next-sitemap）
5. **robots.txt**: 適切なクロール制御
6. **canonical URL**: 重複コンテンツ対策
7. **パンくずリスト**: 構造化データ付きで実装

### 7.2 コンテンツSEO

**【ターゲットキーワード】**

1. **メインキーワード**: 「○○（地域名） キャンピングカー レンタル」
2. **サブキーワード**: 「キャンピングカー レンタル 家族」「キャンピングカー 初心者」
3. **ロングテール**: 「キャンピングカー レンタル 子連れ おすすめ」等

**【コンテンツ施策】**

- FAQページの充実（検索意図に応える）
- 地域情報との関連付け（観光地、RVパーク情報）
- ブログ機能の将来的追加を想定した設計

### 7.3 ローカルSEO

- Googleビジネスプロフィールの登録・最適化
- NAP情報（Name, Address, Phone）の一貫性確保
- 地域名を含むページタイトル・見出し設計
